## G011에서 새로 잠근 것

G011은 제품 코드 변경이 아니라 기존 설치/제거 표면을 다시 몰아서 검증한 단계다.

- shell install dry-run, npm wrapper, Bun wrapper 통과
- config validate 통과
- safe uninstall dry-run, purge preview dry-run 통과
- invalid port, missing `uv`, unsafe purge, unmarked runtime purge 실패 경로 통과
- Pi4 profile은 host tuning을 적용하지 않고 `host_tuning=not_applied`로 보고
- G063에서 같은 Pi4 RC profile 경계를 다시 확인했고, fresh SSH 배포 없이 기존 hardware-stamped evidence와 로컬 non-mutating profile edge를 재검증함
- focused install pytest `17 passed`
- `git diff --check` 통과
- secret scan에서 raw token/fixture secret 출력 없음

중요한 결론: npm/Bun wrapper에서 exchange credential은 argv로 넘기면 안 된다.
package manager가 인자를 먼저 echo할 수 있으므로, 한 줄 설치에서 credential은 환경변수로 넣어야 한다.
